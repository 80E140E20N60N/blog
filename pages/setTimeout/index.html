同步异步以及定时器

# 同步异步以及定时器

# 什么是同步和异步

  1. 同步：代码是一行一行执行的，一行完成了才会跳到下一行。
  2. 异步：某一行代码还没完成，我们可以让它执行着先，但我们可以开始下一行。也就实现了我们所谓的并行。

同步与异步各有优点，同步可以保证执行的顺序，异步可以保证程序的非阻塞。

# 如何理解JS是单线程的以及单线程JS如何实现异步？

  1. 进程与线程
  2. JS运行时环境
    即JS代码运行的环境，在浏览器和Node里都有，以浏览器为例，浏览器是一个多进程的架构。

其中主要关注渲染进程

  1. JS引擎
    在一个JS运行时环境中，JS 代码只在一个线程中执行（浏览器渲染进程的JS引擎线程）， 所以说JS是单线程的。

注：语言本身没有单线程多线程之分，是因为解释这个语言的引擎是单线程的。

浏览器环境上，单线程的js通过浏览器内核多线程实现异步。

# 前端异步场景

  1. 定时器
  2. 网络请求
  3. 时间绑定
  4. ES6 Promise

# 定时器执行原理

  1. 定时任务可能不会按时执行

定时任务是要等同步任务执行完毕后才去执行，加入同步任务耗时很久，那么定时任务就会出现不能按时执行的情况，例下：


const delay =(second)=>{
    let t = +new Date()
    while (true) {
        if(+new Date()-t>=second){
            break;
        }
    }
}

setTimeout(()=>{
    console.log(2); //5秒后打印
},2000)

delay(5000)
