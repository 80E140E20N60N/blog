前端缓存(https://mp.weixin.qq.com/s/q76Ex6EZ6Ez9q2NUp1d3RQ)

前端缓存，是浏览器为了提升网站的加载性能，缩短用户等待时间而采取的措施，浏览器总是想尽量少地向服务器发送请求，能够从自己保存的副本中得到的，就不去麻烦服务器了。

缓存对于用户来说是友好的，而且对大多数用户透明的，普通用户可能最多只是感觉再次进入一个网站时速度变快了而已，再进一步可能某些用户发现一些静态页面断网后还能被访问。

缓存机制：

缓存可以将其理解为我们下载到硬盘的文件。如何感知到资源的变化是缓存更新的关键，这里涉及到缓存的状态变化：

缓存状态有：无缓存、缓存过时/非法、缓存合法。缓存状态的变化，需要浏览器和服务器之间达成某些协议，这些协议由HTTP头部确定及执行

(function(doc, win) {
  var docEl = doc.documentElement

  var resizeEvt = 'orientationchange' in window ? 'orientationchange' : 'resize'

  var recalc = function() {
    var clientWidth = docEl.clientWidth

    if (!clientWidth) return

    docEl.style.fontSize = 16 * (clientWidth / 1920) + 'px'
  }

  if (!doc.addEventListener) return

  win.addEventListener(resizeEvt, recalc, false)

  doc.addEventListener('DOMContentLoaded', recalc, false)
})(document, window)

Canvas是一个标签，它本身只是一个图形容器，必须使用JS脚本来绘制图形

Canvas提供了极为丰富的图像支持，可以支持在图像上绘制图片。

drawImage()方法可以将整张图像或其中的一部分绘制到canvas内的任意位置上。所绘制的图像叫做“源图像”（source image简称s），绘制到的地方叫做“目标canvas”（destination canvas简称d）。绘制时可以选择进行缩放或保持原样。

drawImage()在使用前要图像要已经加载完毕，通常我们会将其放在onload回调函数中以确保这一点。

            arr.push(_random)
        }
    }

el-select 出现不能回显的问题

解决方案： 使用el-select的change事件，在change事件使用一次深拷贝即可实现回显。

问题原因：可能由于数据层级太多的问题，或许是因为原本的form不存在改绑定的值，通过深拷贝复制一次就可以了,例下：

<template>
  <el-select v-model="searchForm.regionCode" placeholder="请选择"  clearable  @change="changeValue">
    <el-option
      v-for="item in competentList"
      :key="item.id"
